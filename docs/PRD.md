# 讀書筆記本 (Book Notes) - 產品需求文件 (PRD)

## 1. 專案概述
- **背景與動機**：現代人閱讀量大，但往往讀完即忘，缺乏一個簡單有效的地方來統一整理與回顧閱讀過的書籍、心得及評分。
- **目標用戶**：所有人（有閱讀習慣，並希望記錄與回顧書籍心得的使用者）。
- **核心價值主張**：提供一個輕量、直覺的平台，讓使用者能輕鬆紀錄書籍資訊、評分與心得，並能快速搜尋過去的閱讀紀錄。

## 2. 功能需求
以下為系統的主要功能清單，以使用者故事（User Story）格式呈現：

1. **紀錄書名**
   - **使用者故事**：作為讀者，我希望能夠輸入並儲存書籍名稱，以便我能明確知道我讀過哪些書。
2. **撰寫心得**
   - **使用者故事**：作為讀者，我希望能夠撰寫並儲存長篇的閱讀心得，以便我在未來能回顧當時的想法與收穫。
3. **評分星星**
   - **使用者故事**：作為讀者，我希望能夠為書籍給予 1 到 5 顆星的評分，以便我能快速直覺地評估自己對該書的喜愛程度。
4. **書籍搜尋**
   - **使用者故事**：作為讀者，我希望能夠透過關鍵字搜尋書名或心得內容，以便我能從眾多紀錄中快速找到特定的書籍筆記。
5. **評分留言**
   - **使用者故事**：作為讀者，我希望能在評分星星之外，加上簡短的評分留言（例如：「神作推薦」、「翻譯待加強」），以便用簡短一句話補充評分原因。

## 3. 非功能需求
- **技術限制**：系統須使用 Flask 作為後端框架，Jinja2 作為前端模板引擎，以及 SQLite 作為資料庫儲存系統。
- **效能考量**：搜尋功能應能快速回傳結果，確保順暢的操作體驗。
- **安全考量**：確保資料庫防範 SQL Injection，並且對使用者輸入進行基本的防 XSS 處理（可直接利用 Jinja2 預設的自動跳脫機制）。

## 4. MVP 範圍 (Minimum Viable Product)
- **Must Have (必須有)**
  - 新增書籍紀錄（書名、心得、星級評分、評分留言）
  - 在首頁瀏覽所有書籍筆記列表
  - 關鍵字搜尋功能
- **Should Have (應該有)**
  - 編輯現有的書籍筆記
  - 刪除錯誤的書籍筆記
- **Nice to Have (可以有)**
  - 依照星級評分高低排序或篩選筆記
  - 多使用者註冊與登入功能（初期 MVP 可先做單機個人版）
